Best Practices & Tips — Read for Fastest & Most Accurate Results

Which mode should I use?

Direct Extraction

Already have a question paper (PDF)? Use this to extract questions exactly as they appear — same text, same options, same order.

Concept Mining

Have a theory/textbook PDF? AI reads the concepts and generates new questions inspired by the content — not copy-paste.

Pure Generation

No PDF? Just type a topic name like Fluid Mechanics and AI generates fresh questions from scratch.

Tips for better & faster generation

Split large PDFs first

If your PDF has 100+ questions, split it into chunks of 20–30 pages before uploading. Smaller PDFs = faster AI response + less hallucination.

Use small batch sizes

Ask for 10–15 questions max per run. Run multiple times to hit your total target. Smaller batches are faster, more accurate, and less likely to get cut off.

Always upload the Exclude CSV

After every run, export the CSV and upload it in "Exclude Already Generated" before the next run. This is the most reliable way to prevent duplicate questions.

Run in multiple short sessions

Instead of 50 at once, do 5 runs of 10. Each run with updated exclude CSV = zero duplicates and higher quality per question.

Error? Wait & retry

If generation fails with a timeout or overload error, wait 30–60 seconds and try again. Gemini API has rate limits that self-reset quickly.

Scanned PDF? AI uses Vision

If your PDF is image-based (scanned paper), the system automatically switches to Gemini Vision mode — no action needed from your side.

Use Custom Instructions

In the config form, add specific instructions like "Focus on numerical problems only" or "Avoid questions about Topic X" to guide the AI.

Always preview before saving

Review all generated questions in the preview panel. Use Edit to fix any LaTeX or wording issues before hitting Save to DB.

Export CSV as backup

Before saving to DB, always export a CSV copy. If something goes wrong, you can re-import via the Upload CSV button without re-generating.

Workflow summary: Split PDF → Upload chunk → Set count to 10–15 → Generate → Preview & edit → Export CSV → Upload as exclude → Repeat for next chunk → Save to DB when done.
